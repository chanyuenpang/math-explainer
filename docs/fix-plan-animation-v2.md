# 动画修复方案 v2

## 问题诊断

### 问题1：边高亮不明显
- **现状**：默认边色 #374151（深灰），高亮色 #3B82F6（蓝色），对比度不够
- **根因**：默认色太深，高亮色不够亮
- **方案**：
  - 默认边色改为 #D1D5DB（浅灰色）
  - 高亮色改为 #3B82F6（亮蓝色）+ 闪烁动画（yoyo repeat）
  - 高亮时 strokeWidth 从 2 增到 4

### 问题2：角度标注无闪烁
- **现状**：直角标记静态显示，没有动画效果
- **根因**：角度弧线没有参与 stepAnimations 系统
- **方案**：
  - 角度弧线添加闪烁动画（opacity yoyo）
  - 高亮时弧线颜色从 #F59E0B 变为 #3B82F6
  - 弧线加粗

### 问题3：讲解文字在下方，视觉分裂
- **现状**：GeometryCanvas 和 StepPlayer 上下分离，用户视线来回跳
- **根因**：两个组件独立渲染，没有合并到同一个画布
- **方案**：
  - 将讲解文字合并到 SVG 画布内部
  - SVG viewBox 加高，底部区域放文字
  - 文字区域在图形下方但仍在 SVG 内部
  - 或者：将讲解文字放在 SVG 下方紧贴，视觉上一体

  根据最佳实践，推荐方案：
  - SVG 图形占上方 60%
  - 讲解文字在 SVG 内部下方 40% 区域
  - 控制按钮也在 SVG 内或紧贴 SVG 下方
  - 整体一个视觉单元
